上面代码有几个地方需要说明：

  1. 递归和迭代都是在lo > hi时才结束，二分查找可以有很多种不同写法，每种写法的条件不太相同(有些是>=有些是>)。我习惯使用lo > hi这种写法。

  2. 关于最后返回的索引值，可以是lo也可以是hi + 1。这里可以简单思考下，最后时候肯定是lo == mid == hi，然后分两种情况，nums[mid] < target或者nums[mid] > target。不管哪种，最后结果都一样，返回lo或者hi + 1。

寻找左边界

左边界就是第一个大于等于目标数字的数，数组可能有重复数字，但必须是有序的。比如[1,2,2,2,3,4]中找2，那么就返回1；用同样的方法我们还可以找到第1个小于目标数字的数。
