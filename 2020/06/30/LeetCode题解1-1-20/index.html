4. Median of Two Sorted Arrays

这道题虽然是一道Hard难度的题目，但想明白了也不是很难。

这道题目要找两个有序数组的中值，对于有奇数个数的数组，中值是第(n / 2)th数字；对于偶数，是第(n / 2 - 1)th和(n / 2)th的算术平均值。这么一转换，这道题目其实就便变成了求两个有序数组合并后第kth个数字是多少，(这里的kth是从0th开始的)如果有偶数个数字，我们就需要求kth和（k + 1）th。

我们再来看看如何求这个kth数字，因为题目要求O(log(m + n))的时间复杂度，因此要使用二分法。下面是一个示意图：

数组的第kth个数字一共有k+1个数字，我们将其分为两半，前半部分有(k + 1) / 2个数字，后半部分有k + 1 - (k + 1) / 2个数字。假设Ax是A[(k + 1) / 2 - 1]，B[y]是B[k + 1 - (k + 1) / 2 - 1]，那么：

  1. 如果A[x] < B[y]，那么A[0 - x]都小于B[y]，因此这部分一定是前kth的数字；
  2. 如果A[x] >= B[y]，那么B[0 - y]都小于等于A[x]，这部分也一定是前kth的数字。

上面是理想情况，实际可能出现数组长度不够(k + 1) / 2或者k + 1 - (k + 1) / 2的情况。这种情况下另一个数组需要比(k + 1) / 2或者k + 1 - (k + 1) / 2更多的元素。比如需要两个数组各提供4个数字，但第一个数组只有3个元素，那么第二个数组就需要提供5个元素。上面说的可能不太清楚直白，看代码就会发现还是比较简单的。

10. Regular Expression Matching

又是一道hard的题目，但不算太难，需要用*，.和字符串匹配另外一组字符串，其中*采用贪婪匹配模式。

假设s是需要匹配的字符串，p是正则表达式字符串，那么我们需要考虑下面几种情况：

  1. s和p都是空的，成功匹配；
  2. s不是空的，p是空的，匹配失败；
  3. s是空的，p不是空的，这种情况需要进一步讨论；
  4. s不是空的，p不是空的，这种情况需要进一步讨论；

下面来分析第3种情况：

  1. 如果p的长度大于1并且p[1] == '*'，这种情况是可能匹配的，需要用p[2 : len - 1]继续分析；
  2. 如果p的长度大于1并且p[1] != '*'，那么肯定不匹配；
  3. 其他情况(比如p的长度小于等于1等)都不匹配。

最复杂的是第4种情况：

  1. 如果p的长度大于1并且p[1] != '*'，那么需要看p[0]和s[0]是否相同，并且要继续匹配s[1 : len - 1]和p[1 : len - 1]；
  2. 如果p的长度大于1并且p[1] == '*'，那么需要看p[0]和s[0]是否相同，并且需要继续匹配s[1 : len - 1]；如果p[0]和s[0]不相同，那么需要匹配s和p[2 : len - 1]。
  3. 如果p的长度等于1，判断s的长度是否为1，并且判断p[0]和s[0]是否相同；
  4. 其他情况都是不匹配。
